## 1. 기술 환경

| 영역 | 스택 |

## 2. 핵심 차별성

### 2-1. 정확성 최우선 응답
- 모르는 내용은 추측하여 답변하지 않음 — 관련 문서가 없으면 "정보를 찾을 수 없습니다"로 안내
- 수집된 학교 공식 문서만을 근거로 답변 생성

### 2-2. 출처 명시
- 모든 답변에 참고 문서명, 링크, 관련 내용 일부를 항상 함께 표시
- 사용자가 원문 문서를 직접 확인할 수 있도록 접근 경로 제공

### 2-3. 학과 특화 문서 커버리지
- 부경대학교 컴퓨터·인공지능공학부 공지사항, 학부 안내, 학사 안내 문서 전체 수집
- 일반 검색엔진으로는 파악하기 어려운 학과 내부 문서까지 검색 가능

### 2-4. 편리한 UI/UX
- 질문 입력 → 답변 + 출처를 한 화면에서 확인
- 자주 묻는 질문 예시 버튼, 로딩/에러 안내 등 학생 친화적 인터페이스

---

## 3. 사용자 문제 정의

### 현재 문제
- 학교 공지사항과 학사 안내 문서가 분산되어 있어 원하는 정보를 찾기 어렵다
- 키워드 검색만으로는 복합적인 질문(예: "복수전공 신청 기간과 절차가 어떻게 돼?")에 답하기 어렵다
- 학과 홈페이지를 직접 탐색하는 데 시간이 많이 든다

### 해결 방향
- 자연어 질문 → 관련 문서 검색 → 문서 기반 답변 생성의 RAG 파이프라인으로 해결
- 답변과 함께 출처를 명시하여 정보의 신뢰성을 확보

## 4. 사용자 흐름

### 4-1. 일반 사용자
- **홈 (`/`)**
  - 서비스 소개 문구 및 질문 입력창
  - 자주 묻는 질문 예시 버튼
- **질의응답 결과**
  - AI 생성 답변 표시
  - 참고 문서명 + 링크 + 관련 내용 일부 출처 카드
  - 답변 로딩 상태 및 오류 안내 UI
- **문서 상세 (`/doc/:id`)**
  - 원문 문서 내용 확인
  - 목록으로 돌아가기

### 4-2. 관리자
- **관리자 대시보드 (`/admin`)**
  - 문서 업로드 및 반영
  - 업로드된 문서 목록 조회, 삭제, 갱신
  - 최신 문서 반영 확인

## 5. 기능 요구사항

### 1순위 — 필수 기능 (MVP)

| ID | 기능명 | 설명 |
|----|--------|------|
| MVP-1 | 학교 문서 수집 | 컴퓨터·인공지능공학부 공지사항, 학부 안내, 학사 안내 문서를 지정된 범위에서 크롤링하여 수집 |
| MVP-2 | 문서 전처리 | 수집 문서에서 본문 텍스트 추출 및 정제, 청크 단위 분할 |
| MVP-3 | 임베딩 및 벡터 검색 | 전처리 문서를 임베딩하여 벡터 DB 저장, 사용자 질문과 유사한 청크 검색 |
| MVP-4 | RAG 질의응답 | 질문 수신 → 관련 문서 검색 → 문서 기반 답변 생성 |
| MVP-5 | 근거 제시 | 답변과 함께 문서명, 링크, 관련 내용 일부 제공. 모르면 추측 금지 |
| MVP-6 | 사용자 웹 UI | 질문 입력창 + 답변 + 출처를 한 화면에서 확인 가능한 웹 인터페이스 |
| MVP-7 | 관리자 문서 업로드 | 관리자가 새 문서를 업로드하고 시스템에 반영하는 기능 |

### 2순위 — 있으면 좋은 기능

| ID | 기능명 | 설명 |
|----|--------|------|
| OPT-1 | 질문 예시 버튼 | 자주 묻는 질문을 버튼으로 제공하여 입력 편의성 향상 |
| OPT-2 | 로딩 및 에러 UI | 응답 생성 중 로딩 표시, 실패 시 안내 문구 노출 |
| OPT-3 | 문서 목록 관리 | 업로드 문서 목록 조회, 삭제, 갱신 기능 |
| OPT-4 | 문서 유형별 필터링 | 질문 유형에 따라 공지·교육과정·안내 문서를 우선 검색 |
| OPT-5 | 최신 문서 반영 확인 | 새 문서가 시스템에 정상 반영되었는지 확인하는 기능 |
| OPT-6 | 문서 미리보기 | 답변 시 참고한 문서의 해당 구절을 미리 보여주는 기능 |

### 3순위 — 후순위 기능

| ID | 기능명 | 설명 |
|----|--------|------|
| POST-1 | 관리자 로그인 고도화 | 단순 비밀번호 방식에서 정식 인증 로그인으로 전환 |
| POST-2 | 학과 범위 확장 | 컴퓨터·인공지능공학부 외 타 학과/단과대 문서로 확장 |
| POST-3 | 답변 히스토리 저장 | 사용자가 이전 질문·답변 내역을 다시 볼 수 있는 기능 |
| POST-4 | 검색 필터링 | 카테고리, 날짜, 문서 유형 등 세부 검색 조건 제공 |
| POST-5 | 답변 평가 기능 | 사용자가 답변의 도움 여부를 평가할 수 있는 기능 |

## 6. MVP 성공 기준

- 문서 수집 → 전처리 → 임베딩 → 질의응답 파이프라인 정상 동작
- 수집된 문서 범위 내 질문에 대해 정확한 답변 생성 (추측 답변 없음)
- 모든 답변에 출처(문서명, 링크, 관련 내용 일부) 표시
- 사용자 웹 UI에서 질문 입력 → 답변 + 출처 확인 가능
- 관리자 문서 업로드 및 반영 기능 정상 동작
- 실 서비스 배포 후 재학생 접근 가능

## 7. 제외 범위 (Non-Goals)

- 컴퓨터·인공지능공학부 외 타 학과 문서 수집 (1학기 기준)
- 사용자 계정 및 로그인 기능
- 답변 히스토리 저장
- 고도화된 관리자 인증 시스템
- 다국어 지원
- 모바일 앱

---

## 8. 핵심 원칙

- **정확성 최우선** — 답변의 정확성이 가장 중요하다. 모르면 모른다고 한다.
- **추측 금지** — 관련 문서가 없거나 불확실한 경우 절대 추측하여 답변하지 않는다.
- **출처 명시** — 사용자가 원문을 직접 확인할 수 있도록 항상 출처를 함께 제공한다.
- **UI/UX 편의성** — 학생이 어렵지 않게 사용할 수 있는 직관적인 인터페이스를 제공한다.
